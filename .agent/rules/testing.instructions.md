# テスト設計（要件駆動テスト）
- すべての要件（FR-x, NFR-x）に対して、テストケースを設計してください
- テストケースは要件の「検証可能な条件」に基づいて作成してください
  - 例：「1秒以内に支出を記録できる」→ パフォーマンステスト（実測値≤1秒）
  - 例：「3タップ以内で支出を記録できる」→ シナリオテスト（タップ数をカウント）

# 要件とテストケースのトレーサビリティ
- テストケースに対応する要件（FR-x）を明記してください
- すべての要件が少なくとも1つのテストケースでカバーされていることを確認してください

# UI仕様に基づくテストケース
- UI仕様で定義された各画面について、テストケースを作成してください：
  - 各入力値パターン（正常系、境界値、異常系）
  - 各操作（ボタン、タップ、スクロール）の結果
  - メッセージ（成功トースト、エラーダイアログ）の表示確認

# テスト自動化
- テストケースは可能な限り自動化してください（ユニットテスト、UIテスト）
- 自動化できない場合のみ、手動テストケースとしてドキュメント化してください
- テスト実行の再現性を確保してください

# 統合テスト
- 複数の要件に関連するシナリオ（例：「支出を記録 → 月別集計を確認」）についても、統合テストケースを準備してください
