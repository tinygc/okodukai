# 要件・構造・UI仕様の確認（実装前の必須条件）
- 実装前に要件ドキュメント、アーキテクチャ、UI仕様の整合性を確認してください
- 実装対象の画面/機能について、以下を確認してください：
  - 該当する要件（FR-x）は何か
  - アーキテクチャ（使用例、データフロー）は何か
  - UI仕様（入力値、操作、メッセージ）は何か
  - データモデル（テーブル、カラム、制約）は何か

# architectureの遵守
- アーキテクチャ設計ドキュメントに従って実装してください
- Layer間の依存関係を守り、Data層へのアクセスはRepositoryを通じてください
- パフォーマンス要件（例：1秒以内に3タップ以内）を設計に組み込んでください

# TDD（テスト駆動開発）の推奨
- t-wada の提唱するTDDプロセスを参考にしてください：https://t-wada.hatenablog.com/entry/2019/12/23/080000
- テストコードは必ず自動化してください
- テスト駆動でコードの質を保証してください

# レビュー体制
- Function/Module 単位の実装完了時に、アーキテクチャ要件との乖離がないことを確認してください
- レビュー指摘への対応は包括的に実施してください  
- 重大な指摘事項がなくなるまで、修正とレビューを繰り返してください

# データベース制約の実装
- データモデルドキュメントで定義された制約（例：10項目制限、NOT NULL制約）を実装してください
- 制約違反時のエラーハンドリングはUI仕様で定義されたメッセージを使用してください
