# 要件定義ドキュメントの作成
- 要件はユーザからヒヤリングを行って明確にしてください
- 要件定義は測定可能で検証可能な形で記述してください
- 機能要件と非機能要件の両方を明確に定義してください
- 要件には優先順位を付けてください

# スコープと制約の明確化（重要）
- データ量制限（例：カテゴリ最大10件、テンプレ最大10件）は明記してください
- ナビゲーション構造（タブ配置など）や設計方針の重要決定は要件レベルで定義してください
- 未決定な制約は「後で決定が必要」と明記し、アーキテクチャ検証時に決定してください
- データの集計方針（例：臨時収入を支出から分離、月次集計方法）を明確にしてください

# 変更管理
- 変更の影響を評価し、必要に応じてドキュメント全体（要件→アーキテクチャ→UI→データモデル）を更新してください
- 複数ドキュメント間の影響波及を防ぐため、変更の影響範囲を検査してください

# トレーサビリティ
- 各要件に一意の識別子を割り当て、設計、実装、テストにわたって追跡できるようにしてください
# 不具合修正時の要件見直し
- 不具合が発生した場合、要件定義を見直し、必要に応じて修正してください
