# おこづかいアプリ - システムテストドキュメント

## テスト概要

**目的**: Phase 1 & 2 実装の動作確認と品質検証  
**対象**: Android実機（Pixel 4a以上推奨）  
**テスト日**: 2026-02-08  
**テスター**: 開発者  

---

## テスト観点

### 1. Happy Path（正常系）
ユーザーが期待通り操作できるか確認

### 2. Edge Cases（境界値）
- 最小値・最大値の入力
- 空入力・null値の処理
- 上限到達時の動作（カテゴリ最大10個など）

### 3. Error Cases（エラーハンドリング）
- 入力値エラー（0以下の金額など）
- バリデーション失敗時のメッセージ
- データベースエラー時の復旧

### 4. NFR検証（非機能要件）
- **NFR-2**: パフォーマンス基準
- **NFR-3**: ユーザビリティ（3タップ以内）

### 5. UI/UX
- 画面遷移がスムーズか
- エラーメッセージが適切か
- データが正確に表示されるか

### 6. バックアップマイグレーション
- 旧バックアップの復元可否
- 未知フィールドを含むバックアップの復元継続
- バックアップ対象外データの再構築
