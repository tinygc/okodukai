# UI仕様書

## 共通ルール
- 初期表示は「支出入力」
- 下部タブ: 支出入力 / 月次サマリ / 管理
- 入力は最短3タップ以内を優先
- 通貨はJPYを前提とし、金額は整数
- 金額は0以下を保存不可
- 戻る操作で未保存の入力がある場合は確認ダイアログ
- 支出の追加/編集は1秒以内、グラフ表示は2秒以内を目標
- ウィジェット入力は5秒以内に完了する導線にする

---

## 画面: 支出入力

### 目的
- 支出を素早く入力する

### 画面構成
- タブ: 通常 / テンプレ
- 入力欄: 金額
- 入力欄: 日付（デフォルト当日）
- 選択欄: カテゴリ / サブカテゴリ（通常タブのみ）
- アクション: 保存

### 通常タブの挙動
- カテゴリ選択後に金額入力
- サブカテゴリはカテゴリ選択後に表示
- 保存で支出を作成

### テンプレタブの挙動
- テンプレボタンを一覧表示
- タップで固定金額とカテゴリで即登録
- 必要に応じて直近使用順で上位表示

### バリデーション
- 金額は必須、0以下は不可
- カテゴリは必須（通常タブのみ）
- サブカテゴリは任意

### エラーメッセージ
- 金額を入力してください
- 0より大きい金額を入力してください
- カテゴリを選択してください
- 保存に失敗しました

### 操作例
- 通常タブ: 「食事」-> 1200円 -> 保存
- テンプレタブ: 「コンビニ 300円」ボタン -> 即時保存

### メッセージ
- 保存完了時に「支出を記録しました」トースト表示
- テンプレタップ時に「記録しました」のトースト表示
- 未保存データがある場合、戻るボタン押下時に「入力内容を破棄しますか？」確認ダイアログ

## 画面: 月次サマリ

### 目的
- 月の予算と支出の状況を把握する

### 画面構成
- 月選択（当月がデフォルト、過去月へ遡れる）
- 予算 / 支出 / 残額のサマリ
- 臨時収入の別枠合計
- 貯金目標進捗カード（繰越残高 / 目標に使える金額 / 達成状況）
- カテゴリ別支出グラフ
- カテゴリ別まとめ（カテゴリごとの合計一覧、全件表示）
- 支出一覧（当月全件スクロール、月日表示、新しい順）
- ナビゲーション: 下部タブを使用

### 再設計方針（モノクロ）
- 強調は色ではなく「サイズ・太字・余白・角丸・線幅」で行う
- 情報の優先順位: 残額 > 予算/支出 > カテゴリ/一覧

### レイアウト構成（上から順）
- 月ヘッダー（YYYY年M月 + 前月/翌月矢印 + 今月に戻る）
- 残額カード（残額ラベル + 大きい数字 + 予算/支出 + 進捗バー）
- カテゴリ別支出（グラフ + Top3 + 一覧リンク）
- 最近の支出（最新5件 + 全件表示リンク）

### タイポ/余白
- 残額数字: 34sp, Bold
- 残額ラベル: 14sp, Medium
- 予算/支出: 14sp, Regular
- セクションタイトル: 16sp, SemiBold
- 一覧行: 15sp, Regular
- リンク文言: 12sp, Medium
- 画面左右パディング: 16dp
- セクション間の縦余白: 16dp
- 残額カード内パディング: 16dp
- 進捗バー高さ: 6dp
- 角丸: 残額カード20dp、通常カード12dp

### 月選択の挙動
- 左右スワイプで月移動（メイン導線）
- 矢印タップでも月移動（補助導線）
- 「今月に戻る」は当月以外のときだけ表示
- 当月は「今月」ラベルを小さく併記
- 空月は予算/支出/残額を0表示

### カテゴリ別グラフ
- Top3のみ表示（その他は「その他」へ集約）
- 表示順は金額の降順
- 0件のときは「データがありません」
- 「一覧を見る ＞」でカテゴリ別一覧へ遷移

### 支出一覧
- 最新5件まで表示、表示順は日付降順
- 未分類は「未分類」固定表示
- 0件のときは「今月の支出はありません」
- 「全件表示 ＞」で支出一覧へ遷移
- 行タップで編集、スワイプで削除（削除前確認ダイアログ）

### インタラクション
- 画面入場: 残額カード/進捗バーを先にフェードイン（150ms）
- グラフ→一覧はスタッガー表示（各80ms）
- 月切替: コンテンツ全体スライド（120ms）
- 数値変化: クロスフェード（150ms）

### 挙動
- 月切替で表示内容を更新
- 月選択で過去月へ遡れる

### エラーメッセージ
- データ取得に失敗しました

### 操作例
- 「今月」をタップして当月に戻る
- 未分類の支出にカテゴリを付けて保存

### メッセージ
- 編集完了時に「支出を更新しました」トースト表示
- 削除前に「この支出を削除しますか？」確認ダイアログ

## 画面: カテゴリ管理

### 目的
- カテゴリとサブカテゴリの管理

### 画面構成
- カテゴリ一覧（階層表示）
  - 各カテゴリをタップするとサブカテゴリ一覧を展開
  - カテゴリごとに追加 / 編集 / 削除
  - ドラッグ&ドロップで並び替え可能
- サブカテゴリはカテゴリ配下に表示
  - サブカテゴリごとに編集 / 削除
  - ドラッグ&ドロップで並び替え可能

### 並び替えの挙動
- 親カテゴリ同士の並び替えが可能
- サブカテゴリは親カテゴリ内で並び替え可能
- ドラッグ開始時にアイテムを視覚的に強調表示
- ドロップ時に新しい並び順を自動保存
- 並び順が未設定のカテゴリは作成日時順で表示

### バリデーション
- カテゴリ名は必須
- サブカテゴリ名は必須
- カテゴリは最大10件まで
- サブカテゴリは各カテゴリごとに最大10件まで

### エラーメッセージ
- カテゴリは最大10件です
- サブカテゴリは最大10件です
- カテゴリ名を入力してください
- サブカテゴリ名を入力してください
- 保存に失敗しました
- 並び順の保存に失敗しました

### 操作例
- カテゴリ「趣味」を追加
- 「趣味」をタップしてサブカテゴリ一覧を表示
- サブカテゴリ「本」を追加
- 「本」を削除
- カテゴリをドラッグ&ドロップで並び替え
- サブカテゴリをドラッグ&ドロップで並び替え

### メッセージ
- 保存完了時に「カテゴリを保存しました」トースト表示
- 削除前に「このカテゴリを削除しますか？」確認ダイアログ
- 並び替え完了時に「並び順を保存しました」トースト表示
